Responsable

Abyssus Bot es un servicio automatizado desarrollado por Mr.Abyssus. Al integrar el bot en tu servidor, el propietario y su equipo de soporte actúan como responsables finales del manejo y visualización de los registros (logs) dentro de su comunidad.

Datos Recolectados y Almacenamiento

Procesamos e indexamos la información mínima técnica estructurada necesaria para el funcionamiento de los módulos interactivos. Esta información se aloja de forma persistente en nuestro clúster de **MongoDB Atlas**:

  • Identificadores de Red: ID único de Discord de usuarios, servidores (guilds) y canales de texto/voz.
  • Perfil Público: Nombre de usuario, tags y metadatos de avatares para renders visuales en el Dashboard.
  • Configuraciones de Servidor: Ajustes de bienvenida, canales asignados, colores hexadecimales y estructuras personalizadas.
  • Módulos Activos: Datos acumulativos de economía virtual (AbyCoins), registros de moderación (modlogs) e indexación de invitaciones en tiempo real.

Uso del Dashboard (Web)

Cuando accedes a nuestro panel de control centralizado, gestionamos credenciales efímeras mediante la API oficial de Discord mediante protocolos seguros:

  • Tokens OAuth2: Claves efímeras de acceso para validar tu identidad de desarrollador o administrador. No almacenamos contraseñas.
  • Sesiones Seguras: Cookies técnicas encriptadas estrictamente necesarias para sostener tu sesión en el panel de control.
  • Mapeo de Permisos: Lectura en tiempo real de la lista de servidores donde posees facultades de gestión (Manage Guild / Administrator).

Seguridad e Infraestructura Cloud

Hemos erradicado el almacenamiento en archivos planos locales (JSON), migrando a un ecosistema en la nube blindado:

  • Cifrado en Tránsito: Toda comunicación entre el servidor Express, el cliente de Discord y la base de datos se realiza mediante TLS/SSL de alta seguridad.
  • Restricción de Acceso: El clúster de MongoDB Atlas cuenta con seguridad a nivel de red (IP Whitelisting) y credenciales encriptadas en variables de entorno (process.env).
  • Aislamiento de Documentos: Los saldos de economía, configuraciones de bienvenida y datos analíticos se organizan mediante colecciones indexadas e independientes por cada ID de servidor.

Control, Modificación y Purga

Garantizamos el derecho de supresión y el control total de los datos mediante herramientas del sistema:

  • Persistencia Dinámica: Las actualizaciones en el Dashboard ejecutan operaciones atómicas (findOneAndUpdate), sobreescribiendo registros antiguos de forma inmediata en caliente.
  • Eliminación Definitiva: Los administradores autorizados pueden purgar las colecciones de datos o configuraciones específicas desde el panel web, eliminando el documento de la base de datos permanentemente.
  • Derecho al Olvido (Purga Automática): Si Abyssus es expulsado de un servidor o este permanece inactivo, los datos vinculados en el clúster se eliminan de forma automatizada tras 90 días de desuso.
