Responsable

Abyssus Bot es un servicio automatizado desarrollado por Mr.Abyssus. Al integrar el bot en tu servidor, el propietario y su equipo de soporte actúan como responsables finales del manejo y visualización de las configuraciones y registros dentro de su comunidad.

Datos Recolectados y Almacenamiento

Procesamos e indexamos la información técnica mínima necesaria para el funcionamiento del sistema de gestión. Esta información se aloja de forma persistente en nuestro clúster de MongoDB Atlas.

Identificadores de Discord: ID único del servidor (guildId) y del canal de texto asignado para las notificaciones.

Configuraciones de Personalización: Parámetros estéticos del mensaje de bienvenida, que incluyen el título del embed, el cuerpo del mensaje, el texto del footer, la URL de la imagen y el color en formato hexadecimal.

Uso del Dashboard (Web)

Cuando accedes a nuestro panel de control centralizado, gestionamos credenciales efímeras mediante la API oficial de Discord a través de protocolos seguros.

Tokens de Acceso: Claves efímeras para validar tu identidad de administrador en el servidor. Nunca almacenamos ni solicitamos contraseñas.

Sesiones Seguras: Cookies técnicas encriptadas estrictamente necesarias para sostener la sesión activa en el panel de control.

Mapeo de Permisos: Lectura en tiempo real de los servidores donde posees facultades de gestión para permitirte modificar la configuración de bienvenida.

Seguridad e Infraestructura Cloud

Hemos migrado el almacenamiento a un ecosistema en la nube protegido para garantizar la disponibilidad y la seguridad de los ajustes del servidor.

Cifrado en Tránsito: Toda comunicación entre tu navegador, el servidor Express y la base de datos de MongoDB Atlas se realiza bajo protocolos encriptados SSL/TLS de alta seguridad.

Restricción de Acceso: El clúster de base de datos cuenta con seguridad a nivel de red (IP Whitelisting) y credenciales protegidas mediante variables de entorno (process.env).

Aislamiento de Configuraciones: Los datos de cada comunidad se organizan de forma independiente mediante un modelo indexado por el ID único de cada servidor.

Control, Modificación y Purga

Garantizamos el derecho de supresión y el control total de los datos almacenados mediante las herramientas del panel y comandos directos.

Actualización Dinámica: Al guardar los cambios en el Dashboard, se ejecuta una operación atómica (findOneAndUpdate) que sobrescribe de manera inmediata los valores antiguos en la base de datos.

Autogestión de Moderación: Todos los datos obtenidos por las herramientas de moderación pueden ser purgados y eliminados de forma definitiva en cualquier momento utilizando el comando /modlogs borrar directamente en tu servidor de Discord.

Eliminación Definitiva: Los administradores autorizados pueden modificar o limpiar los campos del formulario en el panel web, actualizando el documento de forma directa en el clúster.
